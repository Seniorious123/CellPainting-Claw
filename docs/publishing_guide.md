# Publishing Guide

This document summarizes how to publish or share `cellpaint_pipeline_lib` safely.

## Recommended Release Artifact

The recommended artifact is the source release bundle generated by:

```bash
cd /root/pipeline/cellpaint_pipeline_lib
./scripts/build_release_bundle.sh
```

Current bundle shape:

- source tree only
- release docs included
- no provider secrets in repository-managed templates
- no local runtime state in the archive

## Recommended Validation Sequence

Before publishing:

```bash
cd /root/pipeline/cellpaint_pipeline_lib
./scripts/run_release_smoke_test.sh
./scripts/build_release_bundle.sh
```

Review:

- `dist/*.tar.gz`
- `dist/*.sha256`
- `dist/*.manifest.json`

## Secret Hygiene

Do not publish:

- `provider.env`
- Docker `.env`
- OpenClaw runtime `state/`
- local logs with provider metadata
- shell history or screenshots containing API keys

If a key was exposed during development, rotate it before release.

## Config Guidance

For distribution:

- use `configs/project_config.portable.example.json` as the starting point

For the validated current machine:

- use `configs/project_config.example.json`

## OpenClaw Guidance

Two release tracks are maintained:

- AutoDL-like hosts: `integrations/openclaw/autodl/`
- standard Linux hosts with Docker: `integrations/openclaw/docker/`

Both tracks are env-driven and secret-free at the repository template level.

## Platform Limitation on the Current Machine

Docker daemon access is available on the current machine, but full container image build validation is still blocked by platform restrictions.

Observed failure mode during validation:

- BuildKit path: mount permission failure
- classic builder path: `unshare: operation not permitted`

This means:

- the Docker release files are structurally validated
- `docker compose config` resolves successfully
- a full image build still needs a less restricted Linux host for final end-to-end validation

## Recommended External Publication Set

If you want the cleanest publication set, publish:

- the source release bundle from `dist/`
- `README.md`
- `CHANGELOG.md`
- `RELEASE_NOTES.md`
- `docs/first_run_guide.md`
- `docs/release_quickstart.md`
